# 自测脚本 CR 修复记录

## 本次 CR 目标

把第一版自测工具从“可用页面”推进到“更像工程项目”的状态，重点检查真实使用路径、数据质量和后续可维护性。

## 发现并修复的问题

### 1. 模拟考试可能抽到综合题

问题：

综合题不能自动判分，但原来的模拟考试会从全部题型中随机抽题，抽到综合题后流程不完整。

修复：

模拟考试只抽取单选、多选、判断题这类可自动判分题。

### 2. 练习模式重复提交会重复计数

问题：

同一道题答完后，继续点“提交”会不断增加 attempts，影响正确率和错题统计。

修复：

练习模式下，同一轮中同一道题只记录一次提交；重复提交只展示已记录的反馈。

### 3. 未选择答案时无提示

问题：

用户直接点击提交没有任何反馈，容易误以为页面卡住。

修复：

未选择答案时显示明确提示，多选题提示“至少选择一个选项”。

### 4. 回看反馈时选项高亮可能受当前选择影响

问题：

反馈高亮原来依赖当前临时选中状态，回看或重复提交时可能标错。

修复：

反馈高亮改为基于“提交时的答案”。

### 5. 缺少数据质量门禁

问题：

只有解析器，没有独立验证脚本，题库字段错位时不容易第一时间发现。

修复：

新增 `tools/validate-questions.js`，校验题目数量、唯一 ID、题型、答案选项匹配、综合题参考答案等。

### 6. 缺少工程入口

问题：

没有统一命令说明，后续改题库或脚本时容易靠记忆操作。

修复：

新增 `package.json` 和 `README.md`，提供 `npm run parse`、`npm run validate`、`npm run check`。

## 做题流程体验修复记录

### 1. 改为先答题后交卷

原流程是每题点击提交后立即判分、立即显示答案。现在改为：

- 选项点击后只暂存答案。
- 期间可以切换上一题/下一题并修改答案。
- 点击“交卷并看结果”后才统一判分、显示正确答案和解析。

### 2. 支持键盘翻题

新增快捷行为：

- `Enter`：下一题；最后一题时交卷。
- `ArrowRight`：下一题。
- `ArrowLeft`：上一题。
- `A`/`B`/`C`/`D`：选择对应选项。

### 3. 提前交卷不污染错题本

顺序练习、随机练习、错题复习中提前交卷时，未答题只显示为“未作答”，不会计入 attempts，也不会自动加入错题本。模拟考试仍按考试规则处理未答题。

### 4. 综合题自评可更改

综合题在交卷后才能查看参考答案并自评。自评点错后可以重新选择，不会重复增加 attempts。

### 5. 本次验证结果

- `npm run check` 通过。
- 浏览器烟测通过：交卷前不显示答案、答案可修改、Enter 可下一题、返回上一题保留选择。
- 提前交卷验证通过：只记录已答题，未答题不污染错题本。

## 答题卡滚动与 Markdown 渲染修复记录

### 1. 答题卡内部滚动

问题：

题量较多时，右侧答题卡会把页面整体撑长，导致用户需要滚很远才能看完题号。

修复：

- 答题卡面板设置最大高度。
- 题号网格 `attempt-grid` 改为内部滚动。
- 页面主体不再被 172 个题号撑开。

### 2. 解析 Markdown 渲染

问题：

解析中的 Markdown 表格和代码块没有渲染成功，用户看到的是原始 Markdown 文本。

修复：

- 新增轻量 Markdown 渲染器。
- 支持代码块、表格、标题、列表、段落、行内代码、加粗。
- 参考答案和讲评解析改为使用 Markdown 渲染。

### 3. 本次验证结果

- `npm run check` 通过。
- 浏览器验证通过：答题卡为内部滚动，`overflow-y:auto` 生效。
- 浏览器验证通过：参考答案中 Markdown 表格、代码块正常生成，原始代码围栏不再裸露。
