# 自测脚本：成熟题库交互取经记录

本次改造先参考成熟题库/测验系统，再落代码。

## 参考对象

### Moodle Quiz

可借鉴点：

- 支持一页一题或多题布局。
- 有自由/顺序导航方法。
- `Deferred feedback` 用来模拟传统考试：答题期间不立即显示答案，提交后再反馈。
- `Review options` 控制提交后是否显示作答、正确性、反馈、正确答案等。

对应改造：

- 本项目改为“先答题，交卷后统一讲评”。
- 交卷后才显示正确答案和解析。
- 提前交卷时有确认摘要。

参考：[Moodle Quiz settings](https://docs.moodle.org/502/en/Quiz_settings)

### Canvas New Quizzes

可借鉴点：

- Question Navigator 可以跳转到任意题。
- 可 Pin / 标记题目稍后回来。
- 已答题和未答题在导航里有不同状态。

对应改造：

- 增加右侧答题卡。
- 支持题号跳转。
- 支持“标记本题”。
- 答题卡显示当前、已答、标记、错题、未答状态。

参考：[Instructure Canvas New Quizzes Student Guide](https://community.instructure.com/en/kb/articles/661351-how-do-i-take-a-quiz-in-new-quizzes)

### Quizlet Test

可借鉴点：

- 先设置题量和题型。
- Start test 后完成测试。
- Submit test 后查看分数和 review。

对应改造：

- 随机练习和模拟考试题量分离。
- 模拟考试固定 50 题。
- 提交后进入讲评状态。

参考：[Quizlet Studying with Test](https://help.quizlet.com/hc/en-us/articles/360030642972-Studying-with-Test)

## 本次产品决策

### 1. 答题期间不显示答案

原因：

自测和考试模式的关键是保持作答独立性。即时显示答案适合讲解练习，但不适合考试感自测。

### 2. 允许做题中修改答案

原因：

成熟测验系统通常允许回到上一题检查和修改，除非明确设置为顺序不可回退。本项目用于复习，应优先允许修改。

### 3. 增加答题卡

原因：

没有全局导航时，长题组会失控。答题卡能解决“做到哪里了、哪些没答、哪些要回看”的问题。

### 4. 增加标记题

原因：

做题时经常会遇到犹豫题。标记题能让用户先推进，最后集中回看。

### 5. 提交前给摘要

原因：

交卷是高风险动作。必须明确告诉用户已答、未答、标记数量，再确认。

## 本次落地功能

- 右侧答题卡
- 题号跳转
- 标记本题
- 自定义交卷确认弹层
- 答题期隐藏答案
- 讲评期显示正确答案和解析
- 键盘快捷键：
  - `A/B/C/D` 选择选项
  - `Enter` 下一题
  - `ArrowLeft/ArrowRight` 上一题/下一题
  - `F` 标记本题
