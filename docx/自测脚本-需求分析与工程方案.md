# 编译原理自测脚本：需求分析与工程方案

本文定义“要做什么、先不做什么、怎么分阶段做”。  
重点是把项目立住，避免一上来写成临时脚本。

---

## 1. 项目目标

基于当前题库 `编译原理.md`，制作一个本地可用的自测工具。

核心目标：

- 帮用户刷题。
- 帮用户发现薄弱章节。
- 帮用户复盘错题。
- 帮用户考前模拟。

非目标：

- 不做在线平台。
- 不做登录注册。
- 不做多人排行榜。
- 不做复杂云同步。

---

## 2. 用户画像

主要用户：

- 正在准备编译原理考试的学生。
- 已经有题库，但需要更高效刷题。
- 需要从“看答案”变成“真正会做”。

用户痛点：

- Markdown 题库看起来长，容易疲劳。
- 不知道自己哪块薄弱。
- 做错题后没有持续复习机制。
- 综合题不适合单纯选择，需要提示式自评。

## 4. 功能需求

### 4.1 题库导入

第一版：

- 从本地结构化题库 JSON 读取。
- JSON 由脚本从 `编译原理.md` 提取生成。

后续：

- 支持重新解析题库。
- 支持编辑题目标签。

---

### 4.2 做题模式

必须支持：

- 顺序练习：从第 1 题开始做。
- 随机练习：随机抽题。
- 章节练习：只做编译基础/词法/语法/综合题。
- 错题练习：只做历史错题。
- 模拟考试：限时、隐藏解析、最后评分。

---

### 4.3 题目交互

单选题：

- 展示题干、选项。
- 用户选择 A/B/C/D。
- 提交后显示对错、正确答案、解析。
- 记录作答结果。

综合题：

- 展示题干。
- 用户可写简短答案或思路。
- 点击“查看参考答案”。
- 用户自评：不会 / 半会 / 掌握。
- 记录自评结果。

---

### 4.4 错题与复习

必须支持：

- 错题自动加入错题本。
- 答对后可从错题本移除或降低优先级。
- 可按章节筛选错题。
- 可清空记录。

建议支持：

- 连续错 2 次标为“重点错题”。
- 错题列表显示最近错误时间。

---

### 4.5 学习统计

首页显示：

- 总题数
- 已做题数
- 正确率
- 错题数
- 今日完成数量

章节统计：

- 编译基础正确率
- 词法分析正确率
- 语法分析正确率
- 综合题掌握度

---

### 4.6 UI 要求

整体风格：

- 清爽、低干扰、偏学习工具。
- 桌面优先，同时兼容手机。
- 信息密度适中，不做大型营销页。

核心界面：

- 左侧或顶部导航：练习、错题、统计、设置。
- 中间主区域：题目。
- 右侧或下方：进度、题号、状态。

做题页关键状态：

- 未作答
- 已选择未提交
- 回答正确
- 回答错误
- 查看解析
- 下一题

## 6. 技术方案

### 6.1 第一版建议

采用纯前端本地页面：

- `index.html`
- `style.css`
- `app.js`
- `questions.json`

优点：

- 不需要安装环境。
- 打开即可使用。
- 学习记录存在浏览器 `localStorage`。
- 后续容易升级到 Vite/React。

### 6.2 题库解析

建议写一个独立解析脚本：

- 输入：`编译原理.md`
- 输出：`questions.json`
- 负责识别章节、题号、题干、选项、答案、解析。

注意：

- 原始题库里有三段“单选题”，需要用章节上下文区分。
- 综合题没有标准选项，需要单独标记 `type: long_answer`。
- 如果解析失败，要把失败题目输出到日志。

## 8. 风险与处理

### 风险一：Markdown 题库格式不完全统一

处理：

- 解析脚本要保留失败列表。
- 先解析能稳定识别的单选题。
- 综合题可半自动整理。

### 风险二：解析出的答案与题干错位

处理：

- 每题生成唯一 ID。
- 每题保留原始片段 `sourceText` 便于核对。
- M1 阶段输出抽样检查结果。

### 风险三：UI 做得太复杂，影响刷题

处理：

- 第一屏直接进入“继续练习”。
- 统计和设置放次级入口。
- 做题页只保留必要控件。

### 风险四：综合题无法自动判分

处理：

- 第一版采用自评方式。
- 后续可做关键词提示，不做强行机器判分。

## 10. 下一步实施建议

下一步不直接写 UI，先做 M1：

1. 写解析器，把 `编译原理.md` 转成 `questions.json`。
2. 输出解析报告，确认题目数量和字段完整性。
3. 再根据真实 JSON 数据写 UI。

这样做的原因：

- UI 的题型状态依赖数据结构。
- 先解析题库能尽早发现题库格式问题。
- 后面做页面不会被数据质量拖住。
