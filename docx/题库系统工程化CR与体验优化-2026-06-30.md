# 题库系统工程化 CR 与体验优化记录

日期：2026-06-30

## 1. 对标成熟题库后的结论

成熟题库产品的核心不是“点一下立刻判题”，而是把一次作答当成一个可持续、可恢复、可复盘的 attempt。

这次对标后，当前系统应优先补齐这些能力：

1. 作答中自动保存，刷新页面或临时离开后可以恢复。
2. 学生可以先选择答案，交卷前允许反复修改。
3. 交卷前统一确认，明确未答题和标记题。
4. 交卷后集中展示结果、正确答案和解析。
5. 答题卡可滚动，题量大时不挤压主体内容。
6. 答题卡要有清晰状态：当前、已答、标记、错题、未答。
7. 解析内容要可靠渲染 Markdown，尤其是代码块、表格、列表和行内代码。
8. 题库数据需要保持结构化，未来可以向 QTI 等交换标准靠近。

## 2. 本次已经落地的体验修复

### 2.1 自动保存与恢复

问题：此前刷新页面会丢失当前未交卷作答，不像成熟考试系统。

修复：

1. 新增本地 active attempt 存储。
2. 保存当前模式、章节、题目队列、当前位置、草稿答案、标记题和开始时间。
3. 刷新页面后自动恢复未交卷进度。
4. 正式交卷后清除 active attempt，避免下次误恢复旧试卷。
5. 如果只是打开页面但没有任何作答行为，不写入“可恢复草稿”。

学习体验收益：

1. 不怕误刷新。
2. 不怕中途离开。
3. 真实考试感更强。

### 2.2 清除当前答案

问题：选择后只能改成其他选项，不方便把题恢复成“未作答”。

修复：

1. 新增“清除答案”按钮。
2. 单选、判断、简答清空当前答案。
3. 多选清空当前题所有已选项。
4. 清空后答题卡与统计立即同步。

学习体验收益：

1. 交卷前可以主动把拿不准的题留空或重新思考。
2. 答题状态更符合真实试卷逻辑。

### 2.3 答题卡滚动

问题：172 道题时答题卡过高，页面右侧挤压和滚动体验差。

修复：

1. 答题卡主体内部滚动。
2. 统计图例保持在卡片底部区域。
3. 大题量模式下主答题区和答题卡互不干扰。

学习体验收益：

1. 可以快速定位后半部分题目。
2. 不需要整页反复滚动。

### 2.4 Markdown 解析渲染

问题：题库解析里有 Markdown 表格、代码块和行内代码，之前会显示原始反引号。

修复：

1. 支持 fenced code block。
2. 支持 Markdown 表格。
3. 支持段落、标题、列表、行内代码、加粗、斜体。
4. 长表格和代码块保持横向滚动，避免撑破卡片。
5. 表格拆列时忽略行内代码中的竖线，例如 `|α|≤|β|` 不会被误拆成多列。
6. 修复源题库中 Chomsky 文法分类表的残缺单元格，并重新生成结构化题库数据。

学习体验收益：

1. 编译原理题目中的 ACTION/GOTO 表、产生式、项目集能正常阅读。
2. 解析从“能看”升级为“适合复盘”。
3. 公式型表格不会因为竖线符号显示错乱。

## 3. 仍然距离工程化题库的差距

### 3.1 题库导入导出标准化

当前题库已经结构化为 JSON，但还不是通用交换格式。

后续建议：

1. 保留现有 JSON 作为轻量本地格式。
2. 增加导出接口，例如 `export-questions.js`。
3. 长期可参考 QTI 的题目、试卷、评分逻辑、元数据拆分思路。

### 3.2 attempt 历史记录

当前已经能保存“未交卷 attempt”，但完整考试产品还应保存历史 attempt。

后续建议：

1. 每次交卷生成一条历史记录。
2. 记录模式、分数、错题、耗时、章节分布。
3. 支持按历史 attempt 复盘。

### 3.3 复盘过滤器

当前错题复习已经可用，但复盘还可以更细。

后续建议：

1. 只看错题。
2. 只看标记题。
3. 只看未答题。
4. 只看某一章节薄弱题。
5. 只看某一种题型。

### 3.4 学习路径化

当前已经具备练习与模拟考试，但“教会我考高分”的目标还可以继续推进。

后续建议：

1. 根据错题章节自动推荐复习资料。
2. 每次考试后生成“下一轮学习任务”。
3. 高频错题自动进入强化训练。

### 3.5 可访问性与移动端

当前已支持键盘快捷键，但完整工程化还应继续做：

1. 焦点状态更明显。
2. 按钮和答题卡状态增加更完整的 aria 信息。
3. 小屏时答题卡改为抽屉或底部面板。

## 4. 后续推荐路线

第一阶段：把本地自测做到稳定好用。

1. active attempt 自动保存。
2. 答题卡滚动。
3. Markdown 解析。
4. 清除答案。
5. 交卷确认。

第二阶段：把复盘做强。

1. 历史 attempt。
2. 错题按章节归因。
3. 标记题与未答题过滤。
4. 自动生成下一轮复习计划。

第三阶段：把题库工程化。

1. 数据校验继续加强。
2. 导入导出能力。
3. 题库版本记录。
4. 接近 QTI 的结构化元数据。

## 5. 调研依据

1. Moodle Quiz 文档强调 quiz navigation、flag、summary of attempt、submit confirmation、autosave，以及返回后继续作答。
2. Canvas New Quizzes 文档强调 resume quiz、question navigator、pin question、answered/unanswered 状态。
3. 1EdTech QTI 说明了题目和测试在不同应用之间打包、迁移、保持呈现一致性的标准化方向。
