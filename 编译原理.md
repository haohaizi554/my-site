# 编译原理复习题

## 一、单选题

------

## 1. 编译程序与解释程序的根本区别在于？

**题目：**
编译程序与解释程序的根本区别在于（ ）。

**选项：**

- A. 编译程序将源程序翻译成目标程序后执行，解释程序边翻译边执行
- B. 编译程序运行效率低，解释程序运行效率高
- C. 编译程序只能处理高级语言，解释程序只能处理汇编语言
- D. 编译程序不产生目标代码，解释程序产生目标代码

**答案：** A

**解析：**
编译程序会先把整个源程序翻译成目标程序，然后再执行目标程序。解释程序则是一边读取源程序，一边翻译并执行，通常不生成独立的目标程序。因此二者的根本区别是：**编译程序先翻译后执行，解释程序边翻译边执行**。

------

## 2. 一个典型的编译程序通常由几个逻辑阶段组成？

**题目：**
一个典型的编译程序通常由（ ）个逻辑阶段组成。

**选项：**

- A. 4
- B. 5
- C. 6
- D. 7

**答案：** D

**解析：**
按照本题库口径，典型编译程序通常包括 7 个逻辑阶段：

1. 词法分析
2. 语法分析
3. 语义分析
4. 中间代码生成
5. 独立于机器的代码优化
6. 目标代码生成
7. 依赖于机器的代码优化

有些教材会把代码优化合并为一个阶段，因此说成 6 个阶段。但本题库将代码优化拆分为“独立于机器的优化”和“依赖于机器的优化”，所以答案是 **7**。

------

## 3. 在编译过程的各个阶段中，什么是贯穿始终的？

**题目：**
在编译过程的各个阶段中，（ ）是贯穿始终的。

**选项：**

- A. 词法分析
- B. 语法分析
- C. 表格管理，也称符号表管理
- D. 代码优化

**答案：** C

**解析：**
符号表管理贯穿编译的全过程。词法分析阶段需要记录标识符，语法分析阶段需要辅助检查语法结构，语义分析阶段需要查询变量类型、作用域和声明情况，代码生成阶段也需要根据符号表确定变量地址和存储位置。因此符号表管理贯穿始终。

------

## 4. 编译程序的前端与后端通常以什么为界进行划分？

**题目：**
编译程序的“前端”与“后端”通常以（ ）为界进行划分。

**选项：**

- A. 词法分析之后
- B. 语法分析之后
- C. 中间代码生成之后
- D. 目标代码生成之前

**答案：** C

**解析：**
编译前端主要与源语言有关，包括词法分析、语法分析、语义分析和中间代码生成。编译后端主要与目标机器有关，包括代码优化和目标代码生成。中间代码是一种相对独立于源语言和目标机器的表示形式，因此前端和后端通常以**中间代码生成之后**作为分界。

------

## 5. 将编译程序分成前端和后端的主要好处不包括什么？

**题目：**
将编译程序分成前端和后端的主要好处不包括（ ）。

**选项：**

- A. 便于移植到不同目标机器
- B. 便于功能扩充
- C. 极大缩短编译程序的运行时间
- D. 便于开发维护

**答案：** C

**解析：**
前后端划分的主要好处是提高编译器的可移植性、可扩展性和可维护性。例如，如果要支持新的目标机器，主要修改后端；如果要支持新的源语言，主要修改前端。但是前后端划分并不能保证极大缩短编译程序本身的运行时间，因此 C 不是主要好处。

------

## 6. 文法 G 的形式定义中 T 表示什么？

**题目：**
文法 G 的形式定义是一个四元组 G=(V,T,P,S)，其中 T 表示（ ）。

**选项：**

- A. 非终结符集合
- B. 终结符集合
- C. 产生式集合
- D. 开始符号

**答案：** B

**解析：**
文法通常定义为 G=(V,T,P,S)。其中，V 表示文法符号总集合，T 表示终结符集合，P 表示产生式集合，S 表示开始符号。终结符是不能继续推导的基本符号，例如关键字、标识符、运算符等。因此 T 表示终结符集合。

## 7. 上下文无关文法属于几型文法？

**题目：**
在 Chomsky 文法分类中，上下文无关文法属于（ ）型文法。

**选项：**

- A. 0
- B. 1
- C. 2
- D. 3

**答案：** C

**解析：**
Chomsky 文法分类如下：

| 类型 | 名称                      |
| ---- | ------------------------- |
| 0 型 | 无限制文法 / 短语结构文法 |
| 1 型 | 上下文有关文法            |
| 2 型 | 上下文无关文法            |
| 3 型 | 正规文法 / 正则文法       |

上下文无关文法属于 **2 型文法**。

------

## 8. 关于正规文法的描述，正确的是？

**题目：**
下列关于正规文法，也称 3 型文法的描述中，正确的是（ ）。

**选项：**

- A. 产生式右部可以是任意符号串
- B. 产生式形式为 A→aB 或 A→a
- C. 产生式左部必须包含两个符号
- D. 可以产生形如 a^n b^n 的语言

**答案：** B

**解析：**
正规文法又称正则文法，典型的右线性正规文法产生式形式为：

- A → aB
- A → a

其中 A、B 是非终结符，a 是终结符。正规文法只能描述正则语言，不能描述类似 a^n b^n 这种需要数量匹配的语言，因此 D 错误。

------

## 9. 上下文有关文法的产生式必须满足什么？

**题目：**
上下文有关文法，也称 1 型文法的产生式必须满足（ ）。

**选项：**

- A. |α|≤|β|，S→ε 除外
- B. |α|>|β|
- C. 右部必须是终结符
- D. 左部必须是单个非终结符

**答案：** A

**解析：**
上下文有关文法的产生式一般形式为 α→β，要求右部长度不能小于左部长度，即：

|α| ≤ |β|

这表示推导过程中符号串长度通常不能缩短。S→ε 是允许的特殊情况，但通常要求开始符号 S 不出现在任何产生式右部。因此本题选 A。

## 10. 若 S⇒*α，则 α 是什么？

**题目：**
设 G 是一个文法，S 是开始符号，若 S⇒*α，则称 α 是一个（ ）。

**选项：**

- A. 句子
- B. 句型
- C. 产生式
- D. 语言

**答案：** B

**解析：**
如果从开始符号 S 出发，经过 0 步或多步推导得到 α，即 S⇒*α，则 α 称为句型。句型中可以同时包含终结符和非终结符。如果 α 完全由终结符组成，才称为句子。

------

## 11. 文法 G 所描述的语言 L(G) 是什么的集合？

**题目：**
文法 G 所描述的语言 L(G) 是（ ）的集合。

**选项：**

- A. 所有句型
- B. 所有句子
- C. 所有产生式
- D. 所有终结符

**答案：** B

**解析：**
文法 G 所描述的语言 L(G) 是所有句子的集合。句子必须满足两个条件：第一，能够由开始符号 S 推导得到；第二，只由终结符组成。因此 L(G) 不是所有句型的集合，而是所有终结符句子的集合。

------

## 12. 语法树中某子树叶子节点组成的 β 是什么？

**题目：**
在语法树中，若子树的根节点为 A，其所有叶子节点从左到右排列为 β，则 β 相对于 A 是（ ）。

**选项：**

- A. 句柄
- B. 短语
- C. 直接短语
- D. 素短语

**答案：** B

**解析：**
在语法树中，任意一棵子树的叶子节点从左到右排列形成的符号串，称为相对于该子树根节点的短语。题目中子树根节点为 A，叶子节点组成 β，因此 β 相对于 A 是短语。

------

## 13. 一个句型中的最左什么称为句柄？

**题目：**
一个句型中的最左（ ）称为该句型的句柄。

**选项：**

- A. 短语
- B. 直接短语
- C. 素短语
- D. 终结符号

**答案：** B

**解析：**
句柄是自底向上语法分析中的重要概念。一个句型中的最左直接短语称为该句型的句柄。直接短语是由某个非终结符一步推导得到的短语，句柄通常是归约过程中应该优先归约的部分。

------

## 14. 素短语是指不再包含其他什么的短语？

**题目：**
素短语是指至少包含一个终结符且除自身外不再包含其他（ ）的短语。

**选项：**

- A. 句柄
- B. 短语
- C. 素短语
- D. 直接短语

**答案：** C

**解析：**
素短语是短语中的特殊类型。它至少包含一个终结符，并且除自身以外，不再包含其他素短语。注意本题容易误选“短语”或“直接短语”，但按照定义，素短语强调的是不再包含其他**素短语**。

------

## 15. 两个文法生成的语言相同，称为什么？

**题目：**
若两个文法 G1 和 G2 满足 L(G1)=L(G2)，则称 G1 和 G2 是（ ）。

**选项：**

- A. 等价文法
- B. 相同文法
- C. 同构文法
- D. 相似文法

**答案：** A

**解析：**
如果两个文法生成的语言相同，即 L(G1)=L(G2)，则称这两个文法是等价文法。等价文法不要求产生式完全相同，只要求它们能够生成的句子集合完全相同。

------

## 16. 若存在 A⇒+Aα，则文法含有什么？

**题目：**
若文法中存在推导 A⇒+Aα，则称该文法含有（ ）。

**选项：**

- A. 左递归
- B. 右递归
- C. 直接递归
- D. 间接递归

**答案：** A

**解析：**
如果某个非终结符 A 能经过一步或多步推导出以 A 自身开头的符号串，即 A⇒+Aα，则称该文法含有左递归。如果直接存在 A→Aα，则是直接左递归；如果通过多个非终结符间接推出，则是间接左递归。

------

## 17. 最右推导每一步替换什么？

**题目：**
最右推导，也称规范推导的每一步都替换当前句型中的（ ）。

**选项：**

- A. 最左非终结符
- B. 最右非终结符
- C. 最左终结符
- D. 最右终结符

**答案：** B

**解析：**
最右推导是指在每一步推导中，都选择当前句型中最右边的非终结符进行替换。因此最右推导每一步替换的是最右非终结符。与之对应的是最左推导，每一步替换最左非终结符。

------

## 18. 某个句子存在两棵不同语法树，文法是什么？

**题目：**
若一个文法的某个句子存在两棵不同的语法树，则该文法是（ ）。

**选项：**

- A. 无二义性文法
- B. 二义性文法
- C. 递归文法
- D. 非递归文法

**答案：** B

**解析：**
如果一个文法存在某个句子，使得这个句子能够对应两棵或两棵以上不同的语法树，则该文法称为二义性文法。等价地，如果某个句子存在两个不同的最左推导或两个不同的最右推导，也说明该文法具有二义性。

------

## 19. 检查类型不匹配等静态语义错误属于哪个阶段？

**题目：**
在编译过程中，（ ）阶段的主要任务是检查源程序中是否存在类型不匹配等静态语义错误。

**选项：**

- A. 词法分析
- B. 语法分析
- C. 语义分析
- D. 代码优化

**答案：** C

**解析：**
语义分析阶段负责检查程序是否符合语言的语义规则，例如变量是否声明、类型是否匹配、函数参数个数是否正确、数组下标是否合法等。词法分析主要识别单词，语法分析主要检查语法结构，代码优化主要提升程序运行效率，因此本题选 C。

------

## 20. 中间代码生成阶段通常位于什么之后？

**题目：**
编译程序中，中间代码生成阶段通常位于（ ）之后。

**选项：**

- A. 词法分析
- B. 语法分析
- C. 语义分析
- D. 代码优化

**答案：** B

**解析：**
按照本题库口径，中间代码生成通常和语义分析联系在一起，可以合称为“语义分析与中间代码生成”阶段。因此它位于语法分析之后、代码优化之前。严格按照更细的阶段划分，也可以说中间代码生成在语义分析之后，但本题库标准答案是 **语法分析之后**。

------

## 21. 代码优化阶段的主要目标是什么？

**题目：**
代码优化阶段的主要目标是（ ）。

**选项：**

- A. 发现源程序中的语法错误
- B. 缩短目标程序的运行时间或减少空间占用
- C. 将高级语言翻译成汇编语言
- D. 建立符号表

**答案：** B

**解析：**
代码优化是在不改变程序语义的前提下，提高程序运行效率或减少资源消耗。优化目标包括缩短运行时间、减少存储空间、减少指令条数、提高寄存器利用率等。因此本题选 B。

## 22. 目标代码生成器的输入通常是什么？

**题目：**
目标代码生成器的输入通常是（ ）。

**选项：**

- A. 源程序
- B. 单词序列
- C. 语法树
- D. 中间代码

**答案：** D

**解析：**
目标代码生成器通常以中间代码作为输入，将其转换为目标机器上的代码，例如汇编代码或机器代码。中间代码相对独立于具体机器，便于优化和移植。因此目标代码生成器的输入通常是中间代码。

## 23. 错误处理程序通常不能做什么？

**题目：**
编译程序中的错误处理程序通常不能（ ）。

**选项：**

- A. 报告错误信息
- B. 恢复编译工作
- C. 自动修正所有错误
- D. 指出错误位置

**答案：** C

**解析：**
编译程序中的错误处理程序可以报告错误信息、指出错误位置，并尝试恢复编译过程，使编译器能够继续发现后续错误。但是它通常不能自动修正所有错误，尤其是复杂语义错误和程序逻辑错误。因此本题选 C。

## 24. 汇编程序与编译程序的主要区别是什么？

**题目：**
汇编程序与编译程序的主要区别在于（ ）。

**选项：**

- A. 汇编程序处理低级语言，编译程序处理高级语言
- B. 汇编程序不产生目标代码
- C. 编译程序不处理符号
- D. 两者无本质区别

**答案：** A

**解析：**
汇编程序的输入是汇编语言，输出通常是机器语言或目标代码。编译程序的输入通常是高级语言程序，输出是低级语言程序、汇编代码或目标代码。二者的主要区别在于处理对象不同：汇编程序处理低级语言，编译程序处理高级语言。

## 25. 一个典型的编译程序通常包含哪些阶段？

**题目：**
一个典型的编译程序通常包含以下哪些阶段？（ ）

**选项：**

- A. 词法分析
- B. 语法分析
- C. 语义分析与中间代码生成
- D. 代码优化
- E. 目标代码生成

**答案：** ABCDE

**解析：**
典型编译程序通常包括词法分析、语法分析、语义分析与中间代码生成、代码优化、目标代码生成等阶段。词法分析负责把字符流转换成单词序列，语法分析负责检查语法结构，语义分析负责检查类型和含义，中间代码生成负责产生中间表示，代码优化负责提高效率，目标代码生成负责产生目标机器代码。因此五个选项都正确。

## 26. 翻译程序从实现机制角度一般可分为什么？

**题目：**
翻译程序从实现机制的角度一般可分为（ ）。

**选项：**

- A. 编译程序
- B. 解释程序
- C. 汇编程序
- D. 链接程序

**答案：** ABC

**解析：**
翻译程序是将一种语言写成的程序转换为另一种语言形式的程序。常见的翻译程序包括编译程序、解释程序和汇编程序。链接程序主要负责把目标文件和库文件连接成可执行程序，不属于典型的翻译程序。因此本题选 A、B、C。

## 27. 编译程序的前端通常包括哪些？

**题目：**
编译程序的前端通常包括（ ）。

**选项：**

- A. 词法分析
- B. 语法分析
- C. 语义分析
- D. 目标代码生成

**答案：** ABC

**解析：**
编译程序前端主要负责与源语言有关的分析工作，包括词法分析、语法分析和语义分析。目标代码生成依赖目标机器结构，属于后端工作。因此本题选 A、B、C。

## 28. 编译程序的后端通常包括哪些？

**题目：**
编译程序的后端通常包括（ ）。

**选项：**

- A. 中间代码生成
- B. 代码优化
- C. 目标代码生成
- D. 词法分析

**答案：** BC

**解析：**
编译程序后端主要负责与目标机器有关的工作，包括代码优化和目标代码生成。词法分析属于前端，中间代码生成通常归入前端或语义分析与中间代码生成阶段。因此本题选 B、C。

## 29. 关于文法 G=(V,T,P,S)，下列说法正确的有哪些？

**题目：**

- B. 闭包运算 > 连接运算 > 或运算
- C. 或运算 > 连接运算 > 闭包运算

## 4. 消除直接左递归时，A→Aα|β 如何改写？

**题目：**
消除直接左递归时，产生式 A→Aα|β，其中 β 不以 A 开头，可改写为（ ）。

**选项：**

- A. A→βA'，A'→αA'|ε
- B. A→βA'，A'→αA'
- C. A→αA'，A'→βA'|ε
- D. A→β，A'→α

**答案：** A

**解析：**
直接左递归形式为：

```text
A → Aα | β
```

其中 `Aα` 是左递归部分，`β` 是非左递归部分。消除左递归后改写为：

```text
A  → βA'
A' → αA' | ε
```

含义是：先产生一个非左递归开头的 β，然后通过 A' 重复产生若干个 α，最后用 ε 结束。因此答案为 A。

## 5. 提取左因子的主要目的是什么？

提取左因子的主要目的是（ ）。

- A. 消除左递归
- B. 消除回溯，满足 LL(1) 文法条件
- C. 消除右递归
- D. 减少产生式数量

**答案：** B

**解析：**
如果某个非终结符的多个候选式有公共前缀，例如：

```text
A → αβ1 | αβ2
```

自上而下分析时，看到输入符号属于 α 时，无法立即判断应该选择哪一个候选式，可能导致回溯。提取左因子后可改写为：

```text
A  → αA'
A' → β1 | β2
```

这样可以推迟选择，减少或消除回溯，有利于构造 LL(1) 文法。

## 8. 计算 FIRST(α) 时，若 α=X1X2...Xn，且 X1⇒*ε，则如何处理？

- B. FIRST(α)=FIRST(X1)∪FIRST(X2)...，去掉 ε
- C. FIRST(α) 只包含 ε

## 10. LL(1) 文法的充要条件是什么？

**题目：**
文法 G 是 LL(1) 文法的充要条件是（ ）。

- A. 对每个非终结符 A 的任意两个不同产生式 A→α|β，有 FIRST(α)∩FIRST(β)=∅，且若 β⇒*ε，则 FIRST(α)∩FOLLOW(A)=∅
- B. 文法无左递归
- C. 文法无公共左因子
- D. 以上都是必要条件但不充分

**答案：** A

**解析：**
LL(1) 文法要求分析器只看一个向前看符号，就能唯一确定选择哪条产生式。对于同一个非终结符 A 的不同候选式 α 和 β，必须满足：

```text
FIRST(α) ∩ FIRST(β) = ∅
```

如果某个候选式能够推出 ε，还必须满足：

```text
FIRST(其他候选式) ∩ FOLLOW(A) = ∅
```

这些条件保证预测分析表中不会出现多重定义。因此 A 是 LL(1) 文法的充要条件。无左递归和无公共左因子是必要表现，但单独不足以作为完整充要条件。

- C. 是正规文法
- D. 不含 ε 产生式

**答案：** B

**解析：**
如果一个文法能够生成无限多个不同的句子，说明推导过程中必须存在某种可以反复使用的结构。也就是说，某个非终结符经过若干步推导后又能推出包含自身的符号串：

```text
A ⇒+ αAβ
```

这就是递归结构。没有递归的文法只能产生有限多个句子。因此无限语言对应的文法必然含有递归产生式。

## 24. 文法 S→AB，A→a|aA，B→b|bB 生成的语言是什么？

**题目：**
设文法 G 的产生式为：

```text
S → AB
A → a | aA
B → b | bB
```

则该文法生成的语言是（ ）。

**选项：**

- A. {a^n b^n | n≥1}
- B. {a^n b^m | n,m≥1}
- C. {a^m b^n | m,n≥0}
- D. {a^n b^m | n,m≥0}

**答案：** B

**解析：**
先看 A 的推导：

```text
A → a | aA
```

说明 A 可以生成一个或多个 a，即：

```text
a^n，n ≥ 1
```

再看 B 的推导：

```text
B → b | bB
```

说明 B 可以生成一个或多个 b，即：

```text
b^m，m ≥ 1
```

由于：

```text
S → AB
```

所以整个语言为：

```text
{a^n b^m | n,m ≥ 1}
```

它不要求 a 和 b 的数量相等，因此不是 `{a^n b^n | n≥1}`。

## 25. 若 r 和 s 分别是正规式，则 L(r|s) 等于什么？

**题目：**
若 r 和 s 分别是正规式，则 L(r|s) 等于（ ）。

**选项：**

- A. L(r) ∩ L(s)
- B. L(r) ∪ L(s)
- C. L(r) · L(s)
- D. L(r) − L(s)

**答案：** B

**解析：**
正规式中的 `|` 表示或运算，对应语言集合中的并集运算。因此：

```text
L(r|s) = L(r) ∪ L(s)
```

例如 r 表示 `{a}`，s 表示 `{b}`，则 r|s 表示 `{a,b}`。

## 27. LR 分析器的分析表由什么组成？

**题目：**

**选项：**

- A. ACTION 表和 GOTO 表
- B. FIRST 表和 FOLLOW 表
- C. 预测分析表和优先表
- D. 状态转换表和 ε-闭包表

**答案：** A

**解析：**
LR 分析器的分析表由 ACTION 表和 GOTO 表组成：

- ACTION 表：根据当前状态和当前输入终结符决定执行移进、归约、接受或报错操作。
- GOTO 表：根据当前状态和归约后的非终结符决定转移到哪个新状态。

FIRST 和 FOLLOW 主要用于 LL(1) 或 SLR(1) 表构造，预测分析表用于 LL(1) 分析。

## 28. LL(1) 预测分析表不含多重定义，说明文法是什么？

- A. SLR(1)
- B. LR(0)
- C. LL(1)
- D. 算符有限文法

**答案：** C

**解析：**
LL(1) 预测分析表中，如果每个表项最多只含一个产生式，就说明分析器在任何栈顶非终结符和当前输入符号组合下，都能唯一选择产生式，不会发生选择冲突。因此该文法是 LL(1) 文法。

## 31. LL(1) 文法的必要条件包括哪些？

**题目：**

- B. 文法不含公共左因子
- C. 对任意非终结符 A，其各候选式的 SELECT 集两两不相交
- D. 文法必须是二义性的

**答案：** ABC

**解析：**
LL(1) 文法要求只根据一个向前看符号就能唯一选择产生式，因此它必须不含左递归，否则自上而下分析会陷入无限递归；也不能存在导致选择不唯一的公共左因子；并且对于任意非终结符 A，它的各个候选式的 SELECT 集必须两两不相交。LL(1) 文法一定是无二义性的，因此 D 错误。

## 32. 算符优先分析法相对于规范归约法的特点包括哪些？

- A. 归约时不考虑非终结符名，只考虑终结符间的优先关系
- B. 分析速度快
- C. 可能将非规范句型进行归约
- D. 只能用于表达式

**答案：** ABC

**解析：**
算符优先分析法主要根据终结符之间的优先关系进行移进和归约。它在归约时通常不严格考虑具体的非终结符名，而是根据终结符优先关系确定可归约部分，因此分析速度较快。但是它相对于规范归约法不够严格，可能会对非规范句型进行归约。算符优先分析法常用于表达式分析，但不能简单说“只能用于表达式”，因此 D 不选。

## 33. LR(0) 项目根据圆点位置可分为哪些？

**题目：**
LR(0) 项目根据圆点位置可分为（ ）。

**选项：**

- A. 移进项目
- B. 归约项目
- C. 接受项目
- D. 待约项目

**答案：** ABCD

**解析：**
LR(0) 项目根据圆点的位置和圆点后面的符号类型可以分为多类：

1. **移进项目**：圆点后面是终结符，例如 `A→α•aβ`。
2. **待约项目**：圆点后面是非终结符，例如 `A→α•Bβ`。
3. **归约项目**：圆点在产生式右端，例如 `A→α•`。
4. **接受项目**：拓广文法中的特殊项目，例如 `S'→S•`。

因此四个选项都正确。

## 1. 编译程序通常由哪些逻辑阶段组成？简述各阶段的主要任务。

编译程序通常包括以下几个逻辑阶段：

1. **词法分析**
   扫描源程序字符流，识别出关键字、标识符、常数、运算符、界符等单词符号，并输出 Token 序列。
2. **语法分析**
   根据语言的语法规则，分析 Token 序列是否能够构成合法的语句、表达式和程序结构，通常会构造语法树或分析树。
3. **语义分析**
   检查程序是否符合语义规则，例如类型是否匹配、变量是否声明、函数参数是否正确等。
4. **中间代码生成**
   将源程序转换为一种与具体目标机器无关的中间表示，例如三地址代码、四元式、三元式等。
5. **代码优化**
   在不改变程序运行结果的前提下，提高代码质量，例如减少运行时间、减少存储空间、删除冗余代码等。
6. **目标代码生成**
   将中间代码转换为目标机器上的汇编代码或机器代码。
7. **目标代码优化**
   针对具体机器结构进行优化，例如寄存器分配、指令选择、指令调度等。

此外，**符号表管理**和**错误处理**贯穿编译全过程。

## 2. 编译程序的前端和后端一般是如何划分的？这种划分有何优点？

编译程序通常以**中间代码**为界划分为前端和后端。

**前端**主要负责与源语言相关的工作，包括：

- 词法分析
- 语法分析
- 语义分析
- 中间代码生成

**后端**主要负责与目标机器相关的工作，包括：

- 代码优化
- 目标代码生成
- 与机器相关的优化

这种划分的优点包括：

1. **便于移植**
   如果要支持新的目标机器，只需要修改后端，前端可以保持不变。
2. **便于扩展**
   如果要支持新的源语言，可以重新设计前端，后端仍然可以复用。
3. **便于维护**
   前端和后端职责清晰，降低编译程序设计和维护的复杂度。
4. **便于模块化开发**
   不同模块可以由不同人员或团队独立设计，提高开发效率。

## 3. 简述 Chomsky 形式语言体系的四种文法类型，并比较它们的描述能力强弱关系。

Chomsky 文法体系将文法分为四类：

| 类型     | 名称                      | 特点                                                 |
| -------- | ------------------------- | ---------------------------------------------------- |
| 0 型文法 | 短语结构文法 / 无限制文法 | 对产生式限制最少，描述能力最强                       |
| 1 型文法 | 上下文有关文法            | 产生式一般满足 `|α|≤|β|`，S→ε 除外                  |
| 2 型文法 | 上下文无关文法            | 产生式左部是单个非终结符，常用于描述程序语言语法结构 |
| 3 型文法 | 正规文法 / 正则文法       | 产生式形式受限，如 `A→aB` 或 `A→a`，常用于词法分析   |

它们的描述能力从强到弱为：

```text
0 型文法 > 1 型文法 > 2 型文法 > 3 型文法
```

也就是说，正规文法是上下文无关文法的特例，上下文无关文法是上下文有关文法的特例，上下文有关文法又是 0 型文法的特例。

## 4. 词法分析器的主要功能是什么？它输出的单词通常采用什么形式表示？

词法分析器的主要功能是扫描源程序字符流，从中识别出一个个单词符号，例如：

- 关键字
- 标识符
- 常数
- 运算符
- 界符

词法分析器通常会跳过空格、换行、注释等无意义字符，同时能够发现非法字符、非法单词等词法错误。

词法分析器输出的单词通常采用二元组形式表示：

```text
(单词种别，单词属性值)
```

其中：

- **单词种别**表示该单词属于哪一类，例如关键字、标识符、常数等。
- **单词属性值**表示该单词的具体信息，例如标识符在符号表中的位置、常数的值等。

## 5. 简述正规式、正规文法与有限自动机三者之间的关系，并说明它们在词法分析中的作用。

正规式、正规文法和有限自动机三者在描述正规语言方面是等价的。

它们之间的关系如下：

```text
正规式 ⇔ 正规文法 ⇔ 有限自动机
```

具体来说：

1. **正规式**
   用于简洁描述单词符号的模式，例如标识符、整数、关键字等。
2. **正规文法**
   可以用产生式形式描述正规语言。
3. **有限自动机**
   可以根据输入字符进行状态转移，用于识别某个字符串是否属于某个正规语言。

在词法分析中，通常先用正规式描述单词模式，然后将正规式转换为 NFA，再转换为 DFA，最后通过 DFA 实现高效的词法识别。

## 6. 什么是 LL(1) 文法？一个文法成为 LL(1) 文法需要满足哪些条件？

LL(1) 文法是一种适合自上而下预测分析的文法。

LL(1) 中：

- 第一个 `L` 表示从左到右扫描输入串。
- 第二个 `L` 表示使用最左推导。
- `1` 表示只向前看一个输入符号即可决定使用哪条产生式。

一个文法成为 LL(1) 文法通常需要满足：

1. **不含左递归**
   如果存在左递归，自上而下分析可能陷入无限递归。
2. **不含公共左因子**
   如果多个候选式有相同前缀，分析器无法根据一个符号唯一选择产生式。
3. **同一非终结符的不同候选式 SELECT 集两两不相交**
   即对于同一非终结符 A 的不同产生式，分析表中不能出现多重定义。

更形式化地说，若有：

```text
A → α | β
```

则应满足：

```text
FIRST(α) ∩ FIRST(β) = ∅
```

如果某个候选式可以推出空串 ε，还需要满足：

```text
FIRST(其他候选式) ∩ FOLLOW(A) = ∅
```

## 7. 简述自上而下语法分析与自下而上语法分析的基本思想，并各举一种具体方法。

**自上而下语法分析**的基本思想是：

从文法的开始符号出发，根据产生式逐步推导，试图推出与输入串相同的句子。它相当于从语法树的根节点向叶子节点构造语法树。

典型方法：

```text
递归下降分析法、LL(1) 分析法
```

**自下而上语法分析**的基本思想是：

从输入串出发，不断寻找可以归约的部分，将其归约为某个非终结符，最后归约到开始符号。它相当于从语法树的叶子节点向根节点构造语法树。

典型方法：

```text
移进-归约分析法、LR 分析法、算符优先分析法
```

## 8. 什么是句柄？在规范归约中，如何利用句柄进行归约？

句柄是规范归约中的核心概念。按照本题库口径，一个句型中的**最左直接短语**称为该句型的句柄。

句柄具有以下特点：

1. 它是某个产生式的右部。
2. 它是当前规范句型中应该被归约的部分。
3. 规范归约每一步都归约当前句型的句柄。

在规范归约中，若当前句型中出现某个句柄 β，并且存在产生式：

```text
A → β
```

则可以将 β 归约为 A。

例如：

```text
α β γ  ⇒  α A γ
```

其中 β 是句柄，归约后得到更高层次的句型。规范归约是最右推导的逆过程。

## 10. 已知文法 G[E]，对句子 i+i*i 分别给出最左推导和最右推导，并画出语法分析树。

文法为：

```text
E → E + T | T

F → ( E ) | i
```

### （1）最左推导

```text
E
⇒ E + T
⇒ T + T
⇒ F + T
⇒ i + T
⇒ i + T * F
⇒ i + F * F
⇒ i + i * F
⇒ i + i * i
```

### （2）最右推导

```text
E
⇒ E + T
⇒ E + T * F
⇒ E + T * i
⇒ E + F * i
⇒ E + i * i
⇒ T + i * i
⇒ F + i * i
⇒ i + i * i
```

## 12. 构造一个文法 G，使其语言为 L={a^n b^m | n,m≥1，且 n≤m}。

可以构造如下文法：

```text
G = ({S,B}, {a,b}, P, S)
```

产生式集合 P 为：

```text
S → aSb | aB
B → bB | b
```

说明：

1. `S → aSb` 每使用一次，就在左边增加一个 a，在右边增加一个 b，保证 b 的数量至少跟 a 同步增加。
2. `S → aB` 用于结束递归，并保证至少有一个 a。
3. `B → bB | b` 生成一个或多个 b，用于保证 m 可以大于或等于 n。

```text
S ⇒ aB ⇒ ab
S ⇒ aB ⇒ abb
S ⇒ aSb ⇒ aaBb ⇒ aabb
S ⇒ aSb ⇒ aaBbb ⇒ aabbb
```

因此生成的语言为：

```text
{a^n b^m | n,m≥1，且 n≤m}
```

## 15. 已知正规文法 G[S]，求该文法定义的正规式。

文法：

```text
S → aA | bB
A → aS | b
B → bS | a
```

设 S、A、B 分别表示从对应非终结符出发所生成的语言，则有：

```text
S = aA | bB
A = aS | b
B = bS | a
```

代入 A 和 B：

```text
S = a(aS | b) | b(bS | a)
```

展开：

```text
S = aaS | ab | bbS | ba
```

整理：

```text
S = (aa | bb)S | (ab | ba)
```

根据 Arden 定理：

```text
S = (aa | bb)*(ab | ba)
```

所以该文法定义的正规式为：

```text
(aa | bb)*(ab | ba)
```

## 16. 已知正规文法 G[A]，求该文法定义的正规式。

文法：

```text
A → aB | b
B → aA | bB | b
```

设 A、B 表示对应非终结符生成的语言：

```text
A = aB | b
B = aA | bB | b
```

先解 B：

```text
B = bB | aA | b
```

根据 Arden 定理：

```text
B = b*(aA | b)
```

代入 A：

```text
A = aB | b
A = a b*(aA | b) | b
A = a b* a A | a b* b | b
```

整理为：

```text
A = (a b* a)A | (a b* b | b)
```

根据 Arden 定理：

```text
A = (a b* a)*(a b* b | b)
```

所以该文法定义的正规式为：

```text
(a b* a)*(a b* b | b)
```

## 17. 设 Σ={0,1}，正规集 S 由以 00 结尾的所有字符串组成。

### （1）正规式

以 `00` 结尾，前面可以是任意 0、1 串，因此正规式为：

```text
(0 | 1)*00
```

### （2）构造 NFA

构造 NFA：

```text
状态集：{q0, q1, q2}
初态：q0
终态：{q2}
```

转移关系：

| 当前状态 | 输入 0 | 输入 1 |
| -------- | ------ | ------ |
| q0       | q0, q1 | q0     |
| q1       | q2     | ∅      |
| q2       | ∅      | ∅      |

说明：

- q0 可以循环读取任意前缀。
- 当猜测某个 `0` 是倒数第二个字符时，从 q0 转到 q1。
- 再读入一个 `0` 后到 q2。
- 若输入结束时处于 q2，则说明字符串以 `00` 结尾。

### （3）将 NFA 转换为 DFA

对子集进行构造：

| DFA 状态 | NFA 状态集合 | 输入 0 | 输入 1 | 是否终态 |
| -------- | ------------ | ------ | ------ | -------- |
| A        | {q0}         | B      | A      | 否       |
| B        | {q0,q1}      | C      | A      | 否       |
| C        | {q0,q1,q2}   | C      | A      | 是       |

所以 DFA 可表示为：

```text
A --0--> B
A --1--> A

B --0--> C
B --1--> A

C --0--> C
C --1--> A
```

其中：

```text
初态：A
终态：C
```

## 18. 设 Σ={a,b}，正规集 S 由包含子串 ab 的所有字符串组成。

### （1）正规式

包含子串 `ab` 的字符串，可以表示为：

```text
(a | b)*ab(a | b)*
```

### （2）构造 NFA

构造 NFA：

```text
状态集：{q0, q1, q2}
初态：q0
终态：{q2}
```

转移关系：

| 当前状态 | 输入 a | 输入 b |
| -------- | ------ | ------ |
| q0       | q0, q1 | q0     |
| q1       | ∅      | q2     |
| q2       | q2     | q2     |

说明：

- q0 可以读取任意前缀。
- 当读到可能作为子串 `ab` 开头的 `a` 时，转到 q1。
- q1 再读到 `b`，转到 q2。
- q2 是终态，表示已经出现过子串 `ab`，之后任意输入都保持接受状态。

### （3）将 NFA 转换为 DFA

按子集构造法得到：

| DFA 状态 | NFA 状态集合 | 输入 a | 输入 b | 是否终态 |
| -------- | ------------ | ------ | ------ | -------- |
| A        | {q0}         | B      | A      | 否       |
| B        | {q0,q1}      | B      | C      | 否       |
| C        | {q0,q2}      | D      | C      | 是       |
| D        | {q0,q1,q2}   | D      | C      | 是       |

其中：

```text
初态：A
终态：C, D
```

由于 C 和 D 都是接受状态，且行为等价，也可以合并为一个终态 F，得到最小 DFA：

| 状态 | 输入 a | 输入 b | 说明                           |
| ---- | ------ | ------ | ------------------------------ |
| A    | B      | A      | 尚未识别到 ab                  |
| B    | B      | F      | 已看到一个可能作为 ab 开头的 a |
| F    | F      | F      | 已经识别到 ab                  |

其中：

```text
初态：A
终态：F
```

## 19. 已知文法 G[S]，构造项目集规范族、识别活前缀的 DFA，并判断是否为 SLR(1) 文法。

原文法：

```text
S → a A c B e
A → A b | b
B → d
```

### （1）拓广文法

加入新的开始符号：

```text
S' → S
```

编号产生式：

```text
0. S' → S
1. S  → a A c B e
2. A  → A b
3. A  → b
4. B  → d
```

### （2）项目集规范族

```text
I0:
S' → •S
S  → •a A c B e
I1:
S' → S•
I2:
S → a • A c B e
A → •A b
A → •b
I3:
S → a A • c B e
A → A • b
I4:
A → b•
I5:
S → a A c • B e
B → •d
I6:
A → A b•
I7:
S → a A c B • e
I8:
B → d•
I9:
S → a A c B e•
```

### （3）识别活前缀的 DFA

状态转移如下：

| 当前项目集 | 符号 | 下一项目集 |
| ---------- | ---- | ---------- |
| I0         | S    | I1         |
| I0         | a    | I2         |
| I2         | A    | I3         |
| I2         | b    | I4         |
| I3         | c    | I5         |
| I3         | b    | I6         |
| I5         | B    | I7         |
| I5         | d    | I8         |
| I7         | e    | I9         |

### （4）是否存在移进-归约冲突或归约-归约冲突？

不存在冲突。

原因：

- 含有归约项目的状态 I4、I6、I8、I9 中，没有同时存在移进项目。
- 每个归约状态中也没有多个归约项目。
- 因此不存在移进-归约冲突，也不存在归约-归约冲突。

### （5）是否为 SLR(1) 文法？

该文法是 SLR(1) 文法。

因为该文法的 LR(0) 项目集规范族中没有冲突，构造 SLR(1) 分析表时不会出现多重定义。因此它是 SLR(1) 文法。

### （6）构造 FOLLOW 集

```text
FOLLOW(S) = { # }
FOLLOW(A) = { c, b }
FOLLOW(B) = { e }
```

说明：

- `S' → S`，所以 `# ∈ FOLLOW(S)`。
- `S → a A c B e`，A 后面有 c，所以 `c ∈ FOLLOW(A)`。
- `A → A b`，A 后面有 b，所以 `b ∈ FOLLOW(A)`。
- `S → a A c B e`，B 后面有 e，所以 `e ∈ FOLLOW(B)`。

## 20. 已知文法 G[S]，构造项目集规范族、识别活前缀的 DFA，并判断是否为 SLR(1) 文法。

原文法：

```text
S → B B
B → a B | b
```

### （1）拓广文法

加入新的开始符号：

```text
S' → S
```

编号产生式：

```text
0. S' → S
1. S  → B B
2. B  → a B
3. B  → b
```

### （2）项目集规范族

```text
I0:
S' → •S
S  → •B B
B  → •a B
B  → •b
I1:
S' → S•
I2:
S → B • B
B → •a B
B → •b
I3:
B → a • B
B → •a B
B → •b
I4:
B → b•
I5:
S → B B•
I6:
B → a B•
```

### （3）识别活前缀的 DFA

状态转移如下：

| 当前项目集 | 符号 | 下一项目集 |
| ---------- | ---- | ---------- |
| I0         | S    | I1         |
| I0         | B    | I2         |
| I0         | a    | I3         |
| I0         | b    | I4         |
| I2         | B    | I5         |
| I2         | a    | I3         |
| I2         | b    | I4         |
| I3         | B    | I6         |
| I3         | a    | I3         |
| I3         | b    | I4         |

### （4）是否存在移进-归约冲突或归约-归约冲突？

不存在冲突。

原因：

- I4、I5、I6 是归约项目集。
- 这些项目集中没有同时存在移进项目。
- 每个归约项目集中也没有多个归约项目。
- 因此不存在移进-归约冲突，也不存在归约-归约冲突。

### （5）是否为 SLR(1) 文法？

该文法是 SLR(1) 文法。

因为构造出的项目集规范族不存在 LR(0) 冲突，在使用 FOLLOW 集构造 SLR(1) 分析表时也不会出现多重定义。

### （6）构造 FOLLOW 集

```text
FOLLOW(S) = { # }
FOLLOW(B) = { a, b, # }
```

说明：

- `S' → S`，所以 `# ∈ FOLLOW(S)`。
- `S → B B` 中，第一个 B 后面跟着 B，所以 `FIRST(B)={a,b}` 加入 FOLLOW(B)。
- `S → B B` 中，第二个 B 位于产生式末尾，所以 `FOLLOW(S)={#}` 加入 FOLLOW(B)。
- `B → aB` 中，B 位于末尾，只能得到 FOLLOW(B) 本身，不增加新符号。

终结符：

```text
a, b, #
```

非终结符：

```text
S, B
```
